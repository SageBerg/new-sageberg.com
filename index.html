Values

Fun

Craft

  • I enjoy creating software that reduces friction for users. My aim is less confusion and unnecessary actions. I also thoughtfully tend to the health of the code underneath. My goal is to give future developers (myself included) the feeling that someone cared about this code enough to make it navigable, understandable, and change-able.

Responsiveness

  • Humans are always my top priority at work, so I try to start answering questions and reviewing people's code immediately, rather than leaving them hanging. I aim to be effective rather than quick, so I work to provide responses with low ambiguity and avoid putting "Looks good to me"-style responses on non-trivial pull requests. Even if I'm busy, I try to not get in the "I'm too busy to help this person" mindset.
